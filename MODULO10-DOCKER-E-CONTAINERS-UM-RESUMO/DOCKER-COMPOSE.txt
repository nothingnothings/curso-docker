










--> FALANDO DE MULTI-CONTAINER PROJECTS,

APRENDEMOS TAMBEM SOBRE DOCKER-COMPOSE...













A IDEIA DO DOCKER-COMPOSE É SIMPLES----> COM O DOCKER,


PODEMOS RODAR CONTAINERS E CONSTRUIR IMAGES,

MAS 




OS COMANDOS PARA RODAR TUDO ISSO, TODOS OS CONTAINERS,

PODEM FICAR BEM COMPRIDOS...











--> ESPECIALMENTE SE TIVERMOS MÚLTIPLOS ENV VARIABLES,

NETWORKS

E VOLUMES ... --> AÍ FICAREMOS COM COMANDOS DOCKER MTO LONGOS,



E, ESPECIALMENTE EM ENVIRONMENTS COM MÚLTIPOS CONTAINERS,
ISSO 

FICA MT TRABALHO...












--> É AÍ QUE ENTRA O DOCKER-COMPOSE... --> O DOCKER-COMPOSE TE DEIXA 

COLOCAR 


SUA 'CONTAINER LAUNCH CONFIGURATION' 


EM 

1 CONFIGURATION FILE,

QUE 

ENTÃO 

É AUTOMATICAMENTE 'PICKED UP'


QUANDO RODAMOS 

'
docker-compose up',


QUE É O COMANDO QUE VAI INICIALIZAR TODOS OS CONTAINERS...














docker-compose down --> ISSO BRINGS DOWN TODOS NOSSOS CONTAINERS...











--> É POR ISSO QUE O docker-compose 


É UMA EXCELENTE FERRAMENTA PARA CONTAINERS MAIS COMPLEXOS,

PARA PROJETOS MAIS COMPLEXOS (

    e pode ser usada 

    TANTO COM SINGLE-CONTAINER PROJECTS COMO MULTI-CONTAINER PROJECTS..
)
